# ПДн-Сканер — Автоматическое обнаружение персональных данных

Решение хакатона: автоматическое сканирование файлового хранилища и обнаружение файлов, содержащих персональные данные в соответствии с требованиями ФЗ-152.

---

## Быстрый старт

### 1. Установка системных зависимостей

```bash
sudo apt update
sudo apt install -y tesseract-ocr tesseract-ocr-rus
```

### 2. Установка Python-зависимостей

```bash
pip install -r requirements.txt
```

### 3. Запуск

```bash
python edit_edit.py \
  --input-dir /DATA \
  --threads "$(nproc)" \
  --output ~/report_pd_scan.csv
```

Без OCR (быстрее, пропускает изображения и видео):

```bash
python edit_edit.py \
  --input-dir /DATA \
  --skip-ocr \
  --threads "$(nproc)" \
  --output ~/report_pd_scan.csv
```

---

## Параметры запуска

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `--input-dir` | `/DATA` или `$INPUT_DIR` | Путь к директории с файлами |
| `--output` | `report_pd_scan.csv` | Путь к подробному отчёту |
| `--threads` | CPU count | Количество потоков обработки |
| `--skip-ocr` | выключено | Пропустить OCR для изображений и видео |
| `--max-files` | `0` (без лимита) | Ограничить число обрабатываемых файлов |

---

## Выходные файлы

После запуска создаются два файла:

**`result.csv`** — основной файл для системы проверки:

```csv
size,time,name
650666,Sep 27 01:59,customers.csv
1385,Mar 23 13:57,anketa_dms.txt
```

**`report_pd_scan.csv`** — подробный отчёт с категориями ПД, уровнем УЗ и причиной включения.

## Категории ПДн (ФЗ-152)

### Обычные ПД
- Email, телефон
- Паспортные данные (серия + номер)
- СНИЛС (с валидацией контрольной суммы)
- ИНН физлица 12 цифр (с алгоритмической проверкой)
- ИНН юрлица 10 цифр (только с контекстным словом «инн»)
- Номера банковских карт (с проверкой алгоритмом Луна)
- Дата рождения

### Специальные категории (ст.10 ФЗ-152)
- Состояние здоровья
- Биометрические данные
- Религиозные убеждения
- Политические убеждения
- Расовая/национальная принадлежность
- Судимость

## Уровни защищённости (УЗ)

| УЗ | Условие включения |
|----|------------------|
| УЗ-1 | Найдены специальные категории ПД (ст.10 ФЗ-152) |
| УЗ-2 | Банковских карт >= 10 или сильных идентификаторов >= 20 |
| УЗ-3 | Хотя бы один сильный идентификатор или контактов >= 50 |
| УЗ-4 | Обычные ПД в небольшом объёме |

## Как работает детекция

### Быстрый пре-фильтр
Мгновенно отклоняет:
- PDF крупнее 25 МБ (публичные отчёты/книги)
- Изображения меньше 5 КБ (иконки/логотипы)
- Файлы с именами публичных документов (правила, отчёты, протоколы, syllabuses и т.д.)

### Логика по типам файлов

**CSV / Parquet / XLS / JSON:**
Проверяет заголовки колонок — если есть `name`, `email`, `phone`, `birth`, `customer` и т.д., файл включается. Дополнительно учитывается имя файла (`customers.csv`, `incidents.json` и т.п.).

**TXT / MD / HTML:**
Если имя файла указывает на форму с ПД (`anketa`, `zayavka`, `soglasie`, `raspiska`...) и в тексте найден email/телефон/дата/паспорт — включается.

**PDF / DOCX / RTF:**
Включается если: имя файла — форма с ПД + найден контакт/паспорт/дата; или найден паспорт + два контакта; или RTF с именем согласия/выписки.

**Изображения / Видео (OCR):**
OCR извлекает текст (до 3 конфигураций PSM, timeout 60с). Включается если: найден валидный СНИЛС/ИНН/карта; или паспорт + ФИО-маркеры; или дата рождения + ФИО-маркеры.

### Валидация идентификаторов
- **СНИЛС** — проверка контрольных цифр по официальному алгоритму ПФР
- **ИНН** — проверка контрольных цифр (алгоритм ФНС)
- **Банковская карта** — алгоритм Луна

---

## Параллельная обработка

Файлы делятся на три очереди с разными параметрами:

```
Структурированные + текст    -> все потоки,   timeout 30с
PDF / DOCX / RTF              -> потоки/2,     timeout 60с
Изображения / Видео (OCR)    -> CPU-ядра,     timeout 180с
```

Используется `ThreadPoolExecutor` — стабильно работает в WSL и Windows.
